Image duplicates

Try it

A PHP program using AJAX which displays duplicate files according to a supplied xml file.

This was written because my digital camera used to reset itself and I was never sure which photos had been uploaded. Not widely useful but it was easier to use than comparing them manually (and let me play around with xml).

I've put it on here because someone may want an example of xml processing in PHP.

Nonogram Solver Work in progress

Try it

A Nonogram solver using the Codeigniter framework

Written to solve the nonogram for the 2015 GCHQ Christmas card puzzle. It works for other nonograms I've tried too, although the algorithm is far from thorough.

The program uses Ajax with session variables to keep track of the puzzle state.The puzzles must currently be stored as xml files, although I'm working on a graphical editing facility. The Load button will show two puzzles - "gchq" and "umbrella". I've disabled the Save action with it being a public server.

 

The programs were developed primarily for my own interest and probably won't work well on a tablet or smartphone - or even a low-res screen. I may tweak them at some stage to bring them up-to-date but I'm generally more interested in developing new things than revivification. Otherwise this page might be titled "The Running Dead".

It's rather a mixed bag because I'm usually trying to learn how something works or how to make something work. For example, this page is static HTML and CSS (except for the "Back" menu option, even the button clicks use a CSS hack) but the Sudoku helper is essentially all Javascript (except the sidebar).

 

About me

I started programming in the mid '70s, as part of a Computer Studies 'O' level. At first it was coding sheets sent away to be entered into the computer. Then the school installed the technological marvel of a teletype, complete with punched tape facility, connected to the remote mainframe through an acoustic coupler. I was smitten and knew that writing software was what I wanted as a career.

At university, I studied computing science and experienced the limitless joy of having a heroically large stack of punched cards distribute itself over the machine room floor when a rubber band broke. More happily, I also used Unix and wrote programs using a screen for the first time. I even bought a Sinclair ZX80 when they appeared and was able to have my very own computer to explore.

On graduation, I was fortunate enough to get a job with a major software house and have been developing software ever since. I've used over a dozen languages (commercially) through the years and meandered from assembly language to object-oriented web development. I've worked on code to bolt together the wings of an Airbus and code to display its flight plan in the cockpit, worked in refineries and environmental agencies, for multinationals and where I was the only employee. But I still enjoy writing code at home.

I've gone from coding sheets to stylesheets but writing software still thrills and delights. These programs reflect my curiosity and a tendency to explore new (to me) techniques by writing programs for idle amusement.
