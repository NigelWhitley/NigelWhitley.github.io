This applet simulates the sliding tile puzzle you may recall from childhood : one tile has been removed to allow the others to be rearranged.
The code can also be run as an application.

There is general information about the code design here.
More information about the "solving" algorithm and its related code is here.

To try the puzzle, just click on any of the rectangular divisions or "tiles" next to the blank tile : the clicked tile will move to the position occupied by the blank tile. Rather than jumbling the puzzle manually you can click on the "Shuffle" button which will rapidly move tiles at random and then display the resulting layout. Pressing "Shuffle" again will reiterate the process.

You can change the number of rows and columns in the puzzle by clicking on the "Resize" button and entering the new values.
You can also choose the method used to show the tiles of the puzzle by clicking on the "Style" button - one style makes each a Button, another draws the tile using the Graphics functions and the third uses an Image (here it's a photo of the Alps). Be warned that choosing the Image style will cause the the image to be downloaded, which might take a while on a dial-up connection.

When the puzzle is not in its inital or "solved" state, you have the option to let the computer solve the puzzle for you.
  • Pressing "Reset" will immediately return the puzzle to its solved state without showing any intermediate moves.
  • Pressing "Move" will cause a single tile to be moved each time it is pressed until the puzzle is completely solved.
  • Pressing "Solve it" will move tiles using a timer until the puzzle is solved or the user clicks "Stop Solving" or "Reset". The rate at which tiles are moved can be varied by clicking on the "Speed" button and choosing the rate controls accessed by clicking on the "Speed" button.
One point worth making is that the automatic solution is not simply a reversal of the "Shuffle" process. You can shuffle the puzzle manually and then use "Move" or "Solve it" to see how brilliantly the computer manages using its algorithm. Unfortunately, you may well see how stupid it can be too.
